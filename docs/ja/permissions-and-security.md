# 権限とセキュリティ

## 固定セキュリティ制約
- 管理者権限は使用禁止。
- DB 永続化は使用禁止。
- 受領ファイルと生成成果物は一時データとして扱う。

注記:
- 上記の管理者権限禁止は Bot に付与する権限に対する制約。
- コマンド実行者はギルド管理者に限定する。

## Bot 最小必要権限
- `Manage Roles`（必要なロール操作範囲のみ）
- `Manage Channels`
- `View Channels`
- `Send Messages`
- `Attach Files`
- `Read Message History`
- `Use Application Commands`

## 実行者制御
- すべての `/schema` コマンドは実行者のギルド `Administrator` 権限を必須とする。
- `/schema apply` の確認操作は実行者本人に限定。
- Interaction のユーザー ID と元コマンド実行者 ID の一致確認を必須化。

## ファイル取り扱い
- 添付ファイルを受領して解析する。
- 一時データはメモリまたは短命 temp 領域に限定。
- 処理完了後に一時生成物を削除。
- スキーマ、差分、バックアップを DB に保存しない。

## 入力検証
- 適用計画作成前に厳密な YAML 検証。
- 未知キー、未対応チャンネル種別を拒否。
- overwrite 参照と階層参照を適用前に検証。

## 運用安全策
- すべての apply で事前バックアップを必須化。
- すべての apply で事前プレビューを表示。
- すべての変更操作で明示確認を必須化。
- インシデント分析可能な詳細結果を返却。

## 最小監査方針（DBなし）
永続保存が禁止されるため:
- コマンド応答内に操作サマリを残す。
- タイムスタンプ、実行者 ID、結果件数を応答テキストに含める。
- 短期的な監査トレースは Discord メッセージ履歴を利用。
