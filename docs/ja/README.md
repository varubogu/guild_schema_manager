# Discord Server Schema Manager Bot

## 目的
このプロジェクトは、Discord サーバー構成をファイル駆動で管理する Bot の仕様を定義します。

主要ゴール:
- 現在の Discord サーバー構成をファイルとしてエクスポートする。
- アップロードされたファイルとの差分を確認する。
- 明示的なユーザー確認後にのみ構成変更を適用する。

## 非機能制約
- DB は使用しない。
- サーバー側に永続保存しない。
- 管理者権限は使用しない。
- 必要な時だけ Bot を招待して使える。
- Bot は単体プロセスとして動作する。
- コマンド入力/結果として添付ファイルを送受信する。

## MVP 機能
- `roles`、`categories`、`channels` を管理。
- ID と名称の両対応（既定は ID 優先、不一致 guild.id の続行承認時は name 優先）。
- Slash コマンドのみ採用。
- すべての Bot コマンド実行者をギルド管理者に限定。
- 適用前に必ず差分プレビューを表示。
- `/apply` は実行者本人のみ確認ボタンを押せる。
- 適用前バックアップを毎回作成して返却。

## 非対象
- 常時同期するデーモン運用。
- DB ベースの監査履歴保存。
- 確認なしの自動適用。
- 管理者権限の利用。
